POC Summary

This is a "static" POC, of an Angular 15 based replacement for the Swing based desktop application. It's one such POC, and covers specific attributes below. Only a small amount of links/functionality will be simulated.

Attributes

  • A sample Angular Material UI components based layout, with some the common/popular widgets in that library
    • This UI concept attempts to match some of the existing layout, without using MDI (Multiple Document Interface)
    • Note: Colors/fonts are just one example. These are obviously customizable and for graphics designer(s) to improve on.
  • Simulated user login, with just one static user allowed. User: cfo, Password: admin
  • Best practice Angular application components structure (LIFT best practices)
    • See Basics tips for structuring an angular project
  • Multi-stage Dockerfile, which builds same image for all environments
    • Note: An environment variable is passed to the container launch (ex: docker run, or AWS ECS) and respective properties file copied from "assets"" on load (ex: dev, qa, prod).
